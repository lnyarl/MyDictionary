# 백엔드 아키텍처

## 개요

백엔드는 **NestJS**와 **TypeScript**를 사용하여 구축되었으며, Stashy 사전 애플리케이션을 위한 견고하고 확장 가능한 기반을 제공합니다. 이는 의존성 주입, 모듈식 아키텍처 및 포괄적인 오류 처리를 포함한 엔터프라이즈 수준의 패턴을 따릅니다.

## 기술 스택

### 핵심 프레임워크
- **NestJS**: TypeScript를 지원하는 프로그레시브 Node.js 프레임워크
- **TypeScript**: 엄격한 타입 검사 및 최신 JavaScript 기능
- **Knex.js**: 데이터베이스 작업을 위한 SQL 쿼리 빌더
- **PostgreSQL**: 주 관계형 데이터베이스
- **Redis**: 캐싱 및 세션 관리

### 주요 라이브러리
- **Passport.js**: JWT 및 Google OAuth 전략을 사용한 인증 미들웨어
- **class-validator & class-transformer**: 입력 유효성 검사 및 변환
- **nestjs-i18n**: 국제화 지원
- **AWS S3 SDK**: 파일 스토리지 통합
- **Biome**: 린팅 및 포맷팅

## 아키텍처 패턴

### 1. 모듈식 설계

백엔드는 기능 기반 모듈로 구성되며, 각 모듈은 다음을 포함합니다:

```
backend/src/
├── auth/                    # 인증 모듈
│   ├── auth.controller.ts
│   ├── auth.service.ts
│   ├── dto/
│   └── strategies/
├── users/                   # 사용자 관리
│   ├── users.controller.ts
│   ├── users.service.ts
│   ├── users.repository.ts
│   └── dto/
├── words/                   # 사전 단어
│   ├── words.controller.ts
│   ├── words.service.ts
│   ├── words.repository.ts
│   └── dto/
├── definitions/             # 단어 정의
├── likes/                   # 좋아요 기능
├── follows/                 # 사용자 팔로우
├── feed/                    # 활동 피드
├── notifications/           # 사용자 알림
├── badges/                  # 업적 시스템
├── reports/                 # 콘텐츠 보고
└── common/                  # 공유 유틸리티
    ├── database/
    ├── cache/
    ├── guards/
    ├── filters/
    └── events/
```

**이 패턴을 사용하는 이유**: 모듈식 설계는 다음을 제공합니다:
- **명확한 분리**: 각 기능은 자체적으로 포함됩니다
- **유지보수성**: 특정 기능을 찾고 수정하기 쉽습니다
- **테스트 용이성**: 모듈을 독립적으로 테스트할 수 있습니다
- **재사용성**: 공통 기능은 공유 모듈에 있습니다

### 2. BaseRepository를 사용한 리포지토리 패턴

모든 데이터베이스 작업은 `BaseRepository`를 확장하는 리포지토리를 통해 이루어집니다:

```typescript
// BaseRepository는 다음을 제공합니다:
- 소프트 삭제 필터링이 내장된 query() 메서드
- 원자적 작업을 위한 transaction() 지원
- 표준화된 CRUD 작업
- 연결 관리

// 사용 예시:
export class WordsRepository extends BaseRepository {
  async findRecent(limit: number) {
    return this.query()
      .select('*')')
      .orderBy('created_at', 'desc')
      .limit(limit);
  }
}
```

**이 패턴을 사용하는 이유**:
- **일관성**: 모든 데이터베이스 액세스가 동일한 패턴을 따릅니다
- **소프트 삭제**: 소프트 삭제 기능에 대한 내장 지원
- **테스트 용이성**: 테스트에서 리포지토리를 쉽게 모의(mock)할 수 있습니다
- **트랜잭션**: 트랜잭션 관리가 간소화되었습니다
- **보안**: 애플리케이션 전체에서 직접적인 데이터베이스 액세스를 방지합니다

### 3. DTO 기반 유효성 검사

모든 API 엔드포인트는 유효성 검사 데코레이터가 있는 데이터 전송 객체를 사용합니다:

```typescript
export class CreateWordDto {
  @IsString()
  @IsNotEmpty()
  @MaxLength(100)
  term: string;

  @IsString()
  @IsOptional()
  @MaxLength(500)
  description?: string;
}
```

**이 접근 방식을 사용하는 이유**:
- **타입 안전성**: 자동 타입 유효성 검사 및 변환
- **보안**: 경계에서의 입력 정화 및 유효성 검사
- **문서화**: 명확한 API 계약
- **재사용성**: 동일한 DTO를 프론트엔드와 공유할 수 있습니다

### 4. 의존성 주입

NestJS의 의존성 주입은 다음을 위해 전반적으로 사용됩니다:
- **서비스 구성**: 관심사의 명확한 분리
- **구성**: 환경 기반 구성 관리
- **테스트**: 테스트 더블의 쉬운 주입
- **생명 주기 관리**: 인스턴스의 자동 생성 및 정리

## 요청 흐름 아키텍처

```
HTTP 요청
    ↓
글로벌 가드 (JWT 인증, 정지된 사용자)
    ↓
컨트롤러 메서드
    ↓
DTO 유효성 검사 (ValidationPipe)
    ↓
비즈니스 로직 (서비스 계층)
    ↓
데이터 액세스 (리포지토리 계층)
    ↓
데이터베이스 (PostgreSQL/Redis)
    ↓
응답 (글로벌 예외 처리 포함)
```

## 핵심 구성 요소

### 1. 인증 및 권한 부여

**JWT 기반 인증**:
- 사용자 등록/로그인을 위한 Google OAuth 통합
- 세션 관리를 위한 JWT 토큰
- 보안을 위한 갱신 토큰 메커니즘
- 역할 기반 접근 제어

**가드 시스템**:
- `JwtAuthGuard`: 인증이 필요한 경로 보호
- `SuspendedUserGuard`: 정지된 사용자가 보호된 리소스에 액세스하는 것을 방지
- 관리자별 기능을 위한 커스텀 가드

### 2. 데이터베이스 아키텍처

**Knex.js 통합**:
- 타입 안전한 SQL 작업을 위한 쿼리 빌더
- SQL 파일을 사용한 마이그레이션 관리
- 연결 풀링 및 관리
- 트랜잭션 지원

**BaseRepository 패턴**:
```typescript
export abstract class BaseRepository {
  protected query() {
    // 소프트 삭제된 레코드를 자동으로 필터링합니다
    return this.knex('table_name').whereNull('deleted_at');
  }
  
  protected transaction<T>(callback: (trx: Knex.Transaction) => Promise<T>) {
    return this.knex.transaction(callback);
  }
}
```

### 3. 캐싱 전략

**Redis 통합**:
- 세션 저장
- API 응답 캐싱
- 속도 제한
- 실시간 기능 (알림, 피드 업데이트)

### 4. 오류 처리

**글로벌 예외 필터**:
- 중앙 집중식 오류 처리
- 일관된 오류 응답 형식
- 오류 자동 로깅
- 사용자 친화적인 오류 메시지

### 5. 국제화

**다국어 지원**:
- 기본 언어(대체)로 한국어
- 동적 언어 감지
- 헤더 및 쿼리 매개변수 지원
- 모듈별 별도 번역 파일
